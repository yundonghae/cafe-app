# 🌊 동해 카페 (eastsea-cafe)

> 바다를 담은 카페 주문 웹 서비스

깊은 바다의 남색과 파도의 하늘색으로 통일한, 카페 주문 웹 애플리케이션입니다.
고객은 메뉴를 둘러보고 장바구니에 담아 주문하며, 사장님은 별도 관리자 화면에서 메뉴와 주문을 관리합니다.
단순히 색만 파랗게 칠한 것이 아니라, **화면에 들어온 순간 바다를 느끼도록** 물결·떠오르는 카드·빈 화면 일러스트까지 테마를 일관되게 담았습니다.

**배포 주소:** http://eastsea.newlecture.com

---

## ✨ 주요 기능

### 고객

- **메인 페이지** — 움직이는 파도 히어로, 자동 전환 추천 배너(캐러셀), 카테고리 바로가기, 인기 메뉴
- **메뉴 목록/상세** — 카테고리 필터, 이름 검색, 스크롤 시 카드가 떠오르는 연출, 상세 하단 연관 메뉴 추천
- **장바구니** — 수량 조절, 담긴 메뉴 관리
- **주문하기** — 주문 완료 시 주문 코드 발급
- **주문 내역/상세** — 주문 상태 확인, **진행 상태 바(접수 → 제조중 → 완료)** 시각화, 접수 단계만 취소 가능
- **즐겨찾기(찜)** — 메뉴에 하트 저장, 마이페이지에서 모아보기 (품절 메뉴도 찜 가능)
- **마이페이지** — 프로필, 포인트/스탬프, 찜한 메뉴, 최근 주문

### 관리자

- **대시보드** — 전체 주문/매출 요약, **인기 메뉴 랭킹 TOP 5**(취소 주문 제외)
- **메뉴 관리** — 메뉴 CRUD(등록·수정·삭제), 품절 처리, 카테고리 필터·검색
- **주문 관리** — 주문 목록/상세 조회, 상태 변경(접수/제조중/완료/취소)
- **관리자 접근 제어** — 손님 화면에서 진입할 때만 인증, 관리자 내부 이동은 통과

### 🌊 바다 테마 특화 & 접근성

- **움직이는 파도 히어로** — 여러 겹의 SVG 물결이 서로 다른 속도로 흐르고, 맨 앞 겹은 배경색으로 처리해 콘텐츠로 자연스럽게 이어짐
- **스크롤 등장 애니메이션** — 메뉴 카드가 아래에서 물 위로 떠오르듯 순차 등장
- **빈 화면 바다 일러스트** — 장바구니·검색 결과가 비었을 때 조개·유리병 등 바다 모티브 SVG 표시
- **모바일 반응형** — 좁은 화면에서 내비게이션이 햄버거 메뉴로 전환
- **애니메이션 켜기/끄기 토글** — 화면 움직임이 불편한 사용자를 위해 버튼 하나로 전체 애니메이션 제어. 기기의 `prefers-reduced-motion` 설정도 함께 존중
- **색맹 배려** — 주문 진행 단계를 색뿐 아니라 아이콘(✓)·숫자·텍스트로도 구분

## 🛠️ 기술 스택

| 구분        | 기술                                                                            |
| ----------- | ------------------------------------------------------------------------------- |
| 프런트엔드  | HTML / CSS / Vanilla JavaScript (프레임워크 없음)                               |
| 데이터 저장 | localStorage — 메뉴·주문·장바구니·찜 등 브라우저 상태                           |
| 백엔드/DB   | **Supabase (Postgres) — 연결 예정** (현재 localStorage → 클라우드 DB 이전 계획) |
| 스타일      | CSS 변수 기반 바다 팔레트, 콜로케이션(페이지별 CSS) 구조                        |
| 개발 서버   | serve                                                                           |
| 배포        | Nginx 리버스 프록시 + serve (커스텀 도메인)                                     |

> **데이터 저장 관련:** 현재 모든 데이터는 브라우저 localStorage에 저장됩니다.
> 그래서 브라우저마다 데이터가 독립적으로 관리됩니다. 발표 피드백을 반영해
> **Supabase 클라우드 DB로 이전**하는 것을 다음 단계로 계획하고 있습니다.
> 이전이 완료되면 모든 사용자가 같은 데이터를 공유하는 구조가 됩니다.

## 🚀 시작하기

```bash
npm install
npm start
```

http://localhost:3145 에서 확인할 수 있습니다.

### 관리자 접근

관리자 페이지는 손님 화면에서 진입할 때 키를 입력합니다. (세션 동안 유지, 손님 화면 방문 시 해제)

| 구분   | 접근 방법                                  |
| ------ | ------------------------------------------ |
| 관리자 | 헤더의 🔐 관리자 → 키 입력 (`eastsea2026`) |

> ⚠️ 이 키 방식은 실제 보안이 아니라 UX용 접근 제어입니다.
> 키가 브라우저 코드에 노출되므로, 정식 로그인/권한 관리는 Supabase 연결 시 도입할 계획입니다.

## 📁 폴더 구조

```
eastsea-cafe/
├── index.html          # 메인 페이지 (파도 히어로, 추천 배너, 인기 메뉴)
├── menus/              # 메뉴 목록 / 상세 / 연관 메뉴
├── basket/             # 장바구니
├── orders/             # 주문 내역 / 상세 (진행 상태 바)
├── my/                 # 마이페이지 (찜, 최근 주문)
├── admin/              # 관리자 대시보드·메뉴·주문 관리
├── css/                # variables.css (바다 팔레트 · 공통 컴포넌트)
├── js/                 # data.js · utils.js · header.js (공용 데이터/유틸/헤더)
└── favicon.svg         # 바다 테마 파비콘
```

> 화면 코드는 페이지별로 HTML·CSS·JS를 함께 두는 **콜로케이션 구조**이며,
> 색·데이터·공통 함수 같은 공유 자원은 `css/variables.css`, `js/data.js`, `js/utils.js`에 모아 두었습니다.
> 전체 구현 히스토리와 단계별 설계 결정은 `BLUEPRINT.md`에 기록되어 있습니다.

## 🐢 개발하며 겪은 문제 (트러블슈팅)

실제 서버에 배포하면서, 로컬에서는 겪지 못한 문제들을 만났습니다.

1. **로컬에선 되는데 배포 서버에서만 관리자 페이지가 깨짐**
   `serve`가 URL을 자동으로 축약(`admin/index.html` → `admin`)하면서, 상대경로 스크립트가 엉뚱한 파일을 불러온 것이 원인. `serve.json`에서 URL 축약을 비활성화하여 해결.

2. **관리자 인증 방식을 세 번 개선**
   저장 안 함(매번 물음) → localStorage(영구 통과) → referrer 기반(배포 환경에서 실패) → 최종적으로 sessionStorage 기반으로, 손님 화면 방문 시 인증을 해제하는 방식으로 정리.

3. **이미지 응답 코드(200)를 믿을 수 없었던 문제**
   이미지 URL이 정상 응답을 줘도 실제로는 다른 사진인 경우가 있어, 상태 코드가 아닌 실제 이미지를 확인해 교체.

## 🗺️ 앞으로 개선할 점

- **Supabase 연결** — localStorage → Postgres 클라우드 DB로 이전, 모든 사용자가 데이터 공유
- **로그인/회원 기능** — Supabase Auth 기반 회원가입·로그인, 관리자 키 방식을 정식 권한 관리로 교체
- **피드백 반영** — 발표에서 받은 디자인·기능 의견을 우선순위대로 반영
